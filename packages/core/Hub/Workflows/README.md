# Hub Workflows Directory

## Goal
The `Hub/Workflows/` directory contains **Business Organisms**. These are high-level, multi-step Standard Operating Procedures (SOPs) that automate complex business processes.

## What to Place Here
- Markdown workflow definitions (`.md`).
- Processes that involve multiple steps, decisions, or approvals.

## Examples of Tasks & Processes

1.  **Office Manager**: Run `/init-office` (triggering `Hub/Workflows/init-office.md`) to set up digital hub for a new location.
2.  **Finance**: Execute `/report-month` (triggering `Hub/Workflows/monthly-report.md`) to aggregate expenses.
3.  **Sales**: Run `Hub/Workflows/deal-close.md` to trigger the "Won Deal" sequence.
4.  **Project Manager**: Trigger `Hub/Workflows/sprint-planning.md` to review backlog.
5.  **Developer**: Run `Hub/Workflows/feature-branch.md` to automate the git flow.
6.  **HR**: Execute `Hub/Workflows/candidate-screen.md` to parse resumes.
7.  **Marketing**: Run `Hub/Workflows/campaign-launch.md` to check assets.
8.  **Support**: Trigger `Hub/Workflows/ticket-triage.md` to categorize incoming tickets.
9.  **Product**: Run `Hub/Workflows/user-feedback-analysis.md` to aggregate comments.
10. **Legal**: Execute `Hub/Workflows/contract-review.md` to check a document.
