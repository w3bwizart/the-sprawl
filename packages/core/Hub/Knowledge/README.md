# Hub Knowledge Directory

## Goal
The `Hub/Knowledge/` directory stores **Blueprint References**. These are static reference materials, specifications, and guidelines.

## What to Place Here
- Data files (`.json`).
- Reference documents (`.md`).
- e.g., `api-specs.json`, `branding-guidelines.md`.

## Examples of Tasks & Processes

1.  **Developer**: Update `Hub/Knowledge/api-specs.json` when the backend API version changes.
2.  **Designer**: Maintain `Hub/Knowledge/branding-guidelines.md`.
3.  **Sales**: Reference `Hub/Knowledge/pricing-sheet.csv`.
4.  **HR**: Store `Hub/Knowledge/org-chart.json`.
5.  **Legal**: Keep `Hub/Knowledge/terms-of-service.md` updated.
6.  **Product**: Store `Hub/Knowledge/feature-glossary.md`.
7.  **Support**: Reference `Hub/Knowledge/error-codes.json`.
8.  **Marketing**: Store `Hub/Knowledge/personas.md`.
9.  **Finance**: Keep `Hub/Knowledge/exchange-rates.json`.
10. **Operations**: Store `Hub/Knowledge/vendor-list.csv`.
